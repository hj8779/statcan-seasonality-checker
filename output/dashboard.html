Supabase Dashboard → Settings → API. For scanner writes use the service_role key (or GRANT INSERT, UPDATE ON TABLE public.vectors TO anon).

Fetches data from StatCan WDS, computes metadata, and upserts to Supabase. Uses the Supabase connection from the Browse DB tab.

Mode:

How it works: The browser probes vector IDs in batches of 200 via getSeriesInfoFromVector (CORS-safe) and queues valid series into Supabase scan_tasks. The local Node.js worker then picks them up, fetches full data (no CORS restriction), computes metadata, and saves to vectors. Run the worker in a VS Code terminal: npm run worker
